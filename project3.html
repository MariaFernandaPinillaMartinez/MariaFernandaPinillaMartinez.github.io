Project Overview

This project involved comprehensive testing of the car sharing functionality in Urban Routes across multiple browsers and screen resolutions. The focus was on validating both the user interface design and the functional behavior of the booking form, payment methods, and reservation features.

Testing was conducted in two distinct environments: Chrome at 800x600 resolution and Firefox at 1920x1080 resolution. This cross-browser, multi-resolution approach ensured consistent user experience across different devices and viewing conditions. The project emphasized creating comprehensive checklists and test cases to systematically validate every aspect of the car sharing feature.

Key deliverables included a UI design checklist validating layout, typography, and visual elements; functional checklists for payment method and card addition features; test cases for the "Reserve" button and booking functionality; and detailed bug reports in JIRA with screenshots and reproduction steps.

Testing Approach & Methodology

UI Design Testing: Created comprehensive checklist covering layout, spacing, typography, color schemes, button states, and visual consistency. Tested in both Chrome (800x600) and Firefox (1920x1080) to identify resolution-specific issues. Validated navigation flow and modal window behavior.

Functional Testing with Test Design Techniques: Applied equivalence partitioning to input fields to identify valid and invalid input classes. Used boundary value analysis to test edge cases. Created test cases covering both positive scenarios (valid inputs, successful flows) and negative scenarios (invalid inputs, error handling).

Cross-Browser Validation: Executed same test cases in different browsers to identify browser-specific rendering or functional issues. Documented any inconsistencies in behavior or appearance across environments. Ensured consistent user experience regardless of browser choice.

Defect Management: For each bug discovered, created detailed JIRA ticket with: unique ID, descriptive title, step-by-step reproduction, expected vs. actual results, environment details (browser, resolution), severity classification, and supporting screenshots. Maintained traceability between failed test cases and bug reports.

Testing Scope

What was tested: UI design of booking form for selected trip option (e.g., "Luxury"); layout, typography, and visual appearance across resolutions; payment method selection and validation; add card functionality with input validation; "Reserve" button behavior and state changes; booking flow from selection to confirmation; trip cancellation workflow; popup windows and modal interactions.

Out of scope: "Add Driver's License" window design and validation; method of payment window design (tested functionality only); add card window design (tested functionality only); distance and duration calculation accuracy (assumed correct from earlier sprints); timer countdown accuracy.
