Project Overview

This project involved executing comprehensive regression testing for Urban Routes, a transportation application that creates routes and calculates travel duration and pricing for different transport modes. The application features a clean interface with "From" and "To" address fields, three travel modes (Optimal, Flash, and Personal), and icons for various transport types including personal car, walking, taxi, bicycle, scooter, and carpool options.

My primary responsibility was to execute pre-prepared test cases, identify bugs, and document detailed error reports with reproduction steps and severity classifications. This was critical to prevent poor user experiences that could result in negative reviews and loss of customers to competitors.

The testing process involved validating core functionality across multiple transportation modes, verifying map features (relief, satellite, 3D objects), zoom functionality, and ensuring accurate route calculations. Each bug found was meticulously documented in JIRA with clear reproduction steps, expected vs. actual results, and appropriate severity levels.

Testing Approach & Methodology

Test Planning: Reviewed pre-prepared test cases to understand the scope and objectives of regression testing. Identified critical user workflows that needed validation.

Test Execution: Systematically executed each test case following documented steps. Compared actual results against expected outcomes. Noted any deviations or unexpected behaviors.

Bug Reporting: For each failed test case, created a detailed bug report in JIRA including: unique bug ID, descriptive title, step-by-step reproduction instructions, expected vs. actual results, severity classification (Blocker, Critical, Major, Minor, Trivial), and supporting screenshots.

Status Tracking: Maintained test case status (Passed, Failed, Skipped) in Google Sheets with links to corresponding bug reports for failed cases.
